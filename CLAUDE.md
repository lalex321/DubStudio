# DubStudio — контекст проекта

Этот файл — память для будущих сессий Claude Code. Читай при старте работы
в этом репо.

## Что это

Веб-приложение для студии дубляжа. Помощник режиссёра ведёт проекты (фильмы),
импортирует xlsx-файлы с wordcount от Netflix по каждой серии (они приходят
несколько раз — последний файл считается финальным), назначает актёров к
персонажам, смотрит сводные данные, в перспективе — считает зарплаты актёров
по объёму озвученных слов.

## Связь с Pivot

Pivot (`~/Projects/pivot`, `lalex321/pivot`) — stateless-утилита «upload xlsx →
скачать сводный xlsx». Работает в проде, ломать нельзя.

DubStudio — отдельный проект с состоянием. Код НЕ тянуть из Pivot как
зависимость — проще скопипастить парсер xlsx на первый проход, а общий
пакет завести только если/когда появится реальный дубль логики.

Что у Pivot-а полезно списать:
- `consolidator.py` — `PROFILES`, `_FALLBACK_EPISODE_PATTERNS`, логика
  извлечения episode-номера (включая Netflix-форматы `EpisodeN`, `N episode`)
- колонки исходного листа `Word Count Summary`

## Стек (подтверждён пользователем)

- FastAPI + SQLModel + SQLite (Postgres на Render)
- Jinja2 + vanilla JS, без React
- Editable-грид на vanilla JS, autosave на blur/Enter (без Handsontable/ag-Grid)
- Авторизация на сессиях (не HTTP Basic — это для Pivot годится, для
  многопользовательского режима DubStudio — нет)

## Сущности

Старт:
- `User` (email, password_hash, role)
- `Project` (film: title, slug, created_at)
- `Episode` (number, project_id, uploaded_filename, imported_at)
- `Character` (name, project_id)
- `Actor` (name, project_id или глобально)
- `Assignment` (character × actor, project-level, опц. rate)
- `WordCount` (character × episode × metric, значение из xlsx)

Роли: `assistant_director` (основной юзер), `admin`. Расширяем по мере появления
ролевых ограничений.

## MVP-1 (первый рабочий цикл)

1. Один пользователь-админ. Логин/пароль из env (как в Pivot),
   но уже через сессионную cookie, не через HTTP Basic.
2. CRUD проектов (фильмов).
3. В проекте — загрузка xlsx-файлов серий. Файлы сохраняются на диск,
   парсятся в БД. Повторная загрузка той же серии — перезаписывает
   (это и есть «финальный файл по факту озвучки»).
4. Страница проекта: editable-грид «персонаж × серия» + колонка Actor.
   Actor редактируется инлайн, сохраняется через fetch.
5. Без ролей, без расчёта зарплат — это следующий этап.

## Деплой

`render.yaml` уже есть в репо. Render-сервис называется `dubstudio`
(или `dubstudio-<suffix>`, если имя занято). SQLite пока локально; под
Render в какой-то момент переедем на Postgres.

## Конвенции

- Порт локально: `8001` (чтобы не конфликтовать с Pivot на `8000`).
- Тёмная тема с фиолетовыми акцентами (вслед за Pivot).
- Комментарии — только когда «почему» неочевидно. Не описывать «что».
- Не добавлять ненужные фичи/абстракции «на будущее».

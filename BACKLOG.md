удаляем или помечаем ссылкой на коммит.

## Архитектура

- **Client-side xlsx parsing.** Парсить Netflix-xlsx в браузере (SheetJS),
  на сервер слать только JSON с word-count по эпизодам и персонажам.
  Исходный файл никогда не покидает ноутбук помрежа. Это снимает львиную
  долю страхов security-аудита Netflix-адъяcent клиентов, упрощает
  разговор про «какие данные вы храните/обрабатываете», убирает лимит
  на размер upload'а (JSON ~5 КБ/эпизод). Цена: SheetJS ~900 КБ,
  дубликат парсинг-логики на JS, труднее дебажить ошибки парсинга без
  серверных логов. Формат Netflix стабилен годами — дубликат поддерживать
  дёшево.

## Безопасность / TPN-like posture

- Audit log ключевых операций (кто/когда удалил персонажа, слил, изменил WC).
- 2FA для admin-ролей.
- Daily SQLite snapshot → S3/бэкап.
- CSRF-токены на POST-API.
- Rate-limit на `/login`.
- Минимальная длина пароля при создании юзера.

## Поведение

- Rename character (текущее имя → новое, с проверкой уникальности в проекте).
- Delete/unacknowledge для acknowledged-персонажей (сейчас только для красных).

  мерджах (неактуально пока один пользователь).

# DubStudio — бэклог

Неструктурированный список идей и отложенных задач. Когда берём в работу —
удаляем или помечаем ссылкой на коммит.

## Архитектура

- **Client-side xlsx parsing.** Парсить Netflix-xlsx в браузере (SheetJS),
  на сервер слать только JSON с word-count по эпизодам и персонажам.
  Исходный файл никогда не покидает ноутбук помрежа. Это снимает львиную
  долю страхов security-аудита Netflix-адъяcent клиентов, упрощает
  разговор про «какие данные вы храните/обрабатываете», убирает лимит
  на размер upload'а (JSON ~5 КБ/эпизод). Цена: SheetJS ~900 КБ,
  дубликат парсинг-логики на JS, труднее дебажить ошибки парсинга без
  серверных логов. Формат Netflix стабилен годами — дубликат поддерживать
  дёшево. Альтернатива «слать на сервер slim-xlsx с одним листом» —
  отвергнута: всё равно нужен SheetJS для extraction, и сервер остаётся
  с xlsx-парсером (исторически источник CVE), тогда как JSON-парсер
  безопасно прост.

- **Desktop-версия для параноиков.** Упаковка текущего FastAPI+SQLite
  через PyInstaller + pywebview в однофайловый исполняемый бинарник
  (~60–90 МБ). Данные вообще не покидают ноутбук, нулевая сетевая
  активность, работает оффлайн. Каждая студия получает свой инсталлятор
  и свою локальную БД. Цена: отдельные сборки под macOS/Windows,
  $99/год Apple Developer на notarization, собственный updater или
  ручная рассылка новых версий. Minusы multi-user: синхронизация между
  помрежами — либо через общий диск, либо через экспорт/импорт xlsx.

## Безопасность / TPN-like posture

- Audit log ключевых операций (кто/когда удалил персонажа, слил, изменил WC).
- 2FA для admin-ролей.
- Daily SQLite snapshot → S3/бэкап.
- CSRF-токены на POST-API.
- Rate-limit на `/login`.
- Минимальная длина пароля при создании юзера.

## Поведение

- Rename character (текущее имя → новое, с проверкой уникальности в проекте).
- Delete/unacknowledge для acknowledged-персонажей (сейчас только для красных).
- Merge-picker должен предлагать только acknowledged таргеты, либо
  визуально отмечать new-таргеты.
- Merge — гонка по UniqueConstraint Assignment при двух одновременных
  мерджах (неактуально пока один пользователь).

## Продуктово

- Планирование сессий записи, календарь, учёт смен — ждём ответов
  помрежа/владельца и прохода по VoiceQ/Anolla для вдохновения UX.
  Это ключевая фича, превращающая инструмент из «ассистента» в
  «замену второго помрежа» — что кардинально меняет pitch владельцу
  студии (экономия в зарплатах, не в SaaS-фи). Минимум что должно
  уметь:
  - Проверять конфликты и недогрузы актёров в расписании.
  - Подсказывать состав сессии по остатку невозвученных слов/реплик.
  - Выгружать расписание в PDF/xlsx в формате, привычном студии.
  - Хранить историю «кто когда что писал» для расчётов и споров.

  Концепции, подсмотренные у кино-продакшн и ADR-софта (StudioBinder,
  Yamdu, VoiceQ) — всё это уже отлажено индустрией, не переизобретаем:
  - **Stripboard** — список сессий как «полосок» с drag-drop и
    сортировкой. У нас полоска = (дата, время, длительность, актёр,
    студия, список реплик character × episode).
  - **DOOD (Day-Out-Of-Days)** — матрица «актёры × недели» с загрузкой.
    Показывает свободные/перегруженные слоты, основа календарного
    вида.
  - **Call sheet** — печатный «лист на день»: кто, во сколько, куда,
    что пишет. Экспорт в xlsx/PDF, даётся актёрам и администратору
    студии. Уже знакомый нам паттерн экспорта.
  - **Cue sheet** — внутри сессии: список реплик со статусом
    (recorded / pending / skipped). Автоподстановка непокрытых
    (character, episode) для выбранного актёра.
  - **Авто-планировщик** — даём ограничения (актёр доступен Пн/Ср,
    смена 4ч), получаем черновик. На MVP откладываем, стартуем с
    ручного + конфликт-проверки.
  - **Импорт TTAL** (Netflix-формат cue-sheet'ов) — возможно Sofi
    уже получает оттуда реплики или могла бы. Уточнить.

  Открытые вопросы к Sofi/владельцу до проектирования:
  - Бронирование — час / полчаса / смена-4ч / смена-8ч?
  - Одна студия или несколько комнат?
  - Актёр и режиссёр всегда в одной сессии, или помреж пишет одна?
  - Используют ли TTAL от Netflix или только word-count xlsx?
  - Как сейчас решается «актёр не пришёл, переставляем»?
- Расчёт зарплат актёров (по словам / по сменам / комбо).
